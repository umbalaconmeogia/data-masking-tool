# Docker test environment

Disposable MySQL and PostgreSQL servers preloaded with the `companies` /
`employees` test data, for trying the masking tool against real servers.

## Start

```bash
cd docker
docker compose up -d
```

* MySQL 8 on `127.0.0.1:13306` (db `masking_test`, user `root` / `secret`)
* PostgreSQL 16 on `127.0.0.1:15432` (db `masking_test`, user `postgres` / `secret`)

On the **first** start, the init scripts (`mysql-init/01-demo.sql`,
`pgsql-init/01-demo.sql`) create the schema and import the shared CSV data
from `test-data/companies.csv` and `test-data/employees.csv` (10 companies +
100 employees) — MySQL via `LOAD DATA INFILE`, PostgreSQL via `COPY`. The same
CSVs are used by the Python integration tests, so all three databases are
seeded from one data source. Everything under `test-data/`, `mysql-init/` and
`pgsql-init/` is generated by `script-python/tools/generate_test_data.py`.

## Run the masker

```bash
cd ../script-python
pip install -r requirements.txt

# Dry run first (no changes written)
python -m masking_tool --env ../docker/.env.mysql-test --rule ../masking-rule.example.csv --dry-run

# Real run, with a backup mapping of original values
python -m masking_tool --env ../docker/.env.mysql-test --rule ../masking-rule.example.csv --backup masking-backup.csv

# Same against PostgreSQL
python -m masking_tool --env ../docker/.env.pgsql-test --rule ../masking-rule.example.csv --dry-run
```

## Spot-check

```bash
docker compose exec mysql mysql -uroot -psecret masking_test \
  -e "SELECT id, name, name_kana, email, birthday, tel FROM employees LIMIT 5;"

docker compose exec pgsql psql -U postgres masking_test \
  -c "SELECT id, name, name_kana, email, birthday, tel FROM employees LIMIT 5;"
```

## Reset to unmasked data

The init scripts only run on first start, so wipe the volumes to reload:

```bash
docker compose down -v
docker compose up -d
```
